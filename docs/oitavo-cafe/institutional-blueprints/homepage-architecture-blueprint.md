## 1. Sumário Executivo

Este blueprint define a arquitetura estratégica da homepage do Oitavo Café, uma boutique de growth marketing com posicionamento challenger/anti-establishment no mercado brasileiro.

### Decisões-Chave

| Dimensão | Decisão | Rationale |
|----------|---------|-----------|
| **Abordagem Estratégica** | Provocação Challenger | Diferenciação através de ataque ao status quo (agências tradicionais) |
| **Stakeholder Primário** | Empresários/Gestores (30-50 anos) | ICP frustrado com agências tradicionais, busca alternativa |
| **Trust Signals** | Resultados + Transparência Radical | Prova social específica (números) + anti-opacidade |
| **Seções** | 7 seções | Adequado para B2B + Growth + Medium trust |
| **Intensidade Challenger** | 20-100% (calibrada por seção) | Provocação máxima onde suporta, credibilidade onde necessário |

### Proposta de Valor em 5 Segundos

> "Boutique de growth marketing para empresários cansados de pagar caro por resultados que não vêm."

## 5. Inteligência de Credibilidade Setorial

### Padrões do Setor (Agências/Boutiques Marketing)

| Padrão | Frequência | Oitavo Café |
|--------|------------|-------------|
| Logos de clientes | 90%+ | ✅ Incluir (mínimo 4 acima do fold) |
| Case studies | 85%+ | ✅ Testemunhos com outcomes |
| Processo transparente | 60% | ✅ Diferenciador - ir além |
| Preços visíveis | 15% | ✅ **DIFERENCIADOR FORTE** |
| Provocação anti-status-quo | 20% | ✅ Posicionamento único |

### Gaps Competitivos Explorados

1. **Transparência de Preços** - 85% das agências escondem. Oitavo mostra.
2. **Resultados Específicos** - Maioria fala de "soluções". Oitavo fala de "R$16M gerados".
3. **Posicionamento Anti-Agência** - Ninguém ataca o próprio setor. Oitavo ataca.
4. **Acessibilidade Boutique** - Grandes vendem "estrutura". Oitavo vende "parceria".

### Benchmark de Credibilidade

| Métrica | Benchmark Setor | Meta Oitavo Café |
|---------|-----------------|------------------|
| Trust signals above fold | 2-3 | 4+ (logos + números) |
| Testimonials com números | 30% | 100% |
| Processo visível | Raro | Padrão |
| Preço estimado visível | Muito raro | Presente |

└─────────────────────────────────────────────────────────┘

## 8. Arquitetura Seção por Seção

### Seção 1: Hero (Acima do Fold)

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #1 - Primeira impressão |
| **Propósito** | Identidade + Provocação + CTA único |
| **Intensidade Challenger** | 80% |
| **Stakeholder Primário** | Empresário frustrado |

#### Elementos Obrigatórios (Above the Fold)

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Provocativo, específico, 6-10 palavras |
| **Exemplo** | "Cansado de pagar caro por marketing que não funciona?" |
| **Subheadline** | Value prop focada em OUTCOME |
| **Exemplo** | "Boutique de growth que entrega resultados, não relatórios bonitos" |
| **CTA Primário** | ÚNICO, ação clara |
| **Exemplo** | "Diagnóstico Gratuito" ou "Quero Resultados Reais" |
| **Trust Signals** | Mínimo 4 logos + 1-2 números |
| **Exemplo** | Logos clientes + "47 empresas já cresceram conosco" |

#### Anti-Padrões a Evitar

- ❌ "Somos uma agência de marketing digital" (genérico)
- ❌ "Soluções inovadoras para seu negócio" (jargão)
- ❌ Múltiplos CTAs competindo
- ❌ Parede de texto

#### Stanford Guidelines Atendidos

- #1 Precisão (números visíveis)
- #2 Organização real (identidade clara)
- #6 Design profissional

#### Conexão com Próxima Seção

Hero provoca → Seção 2 valida a dor mencionada

---

### Seção 2: Problema/Validação

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #2 - Logo após hero |
| **Propósito** | Validar a dor, criar identificação profunda |
| **Intensidade Challenger** | 100% (máxima provocação) |
| **Stakeholder Primário** | Empresário frustrado |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Articular a dor com precisão |
| **Exemplo** | "Você reconhece esse ciclo?" |
| **Lista de Dores** | 3-5 bullets específicos |
| **Exemplo** | "Relatórios lindos, resultados inexistentes" / "Reuniões intermináveis sobre 'estratégia'" / "Métricas de vaidade que não pagam as contas" |
| **Validação** | Frase que mostra entendimento |
| **Exemplo** | "Nós também ficamos frustrados. Por isso criamos outra coisa." |

#### Tom e Voz

- Irreverente mas empático
- Atacar a INDÚSTRIA, não o cliente
- "Nós entendemos porque também odiamos isso"

#### Stanford Guidelines Atendidos

- #3 Expertise (conhecimento profundo do problema)
- #4 Pessoas confiáveis (empatia genuína)

#### Conexão com Próxima Seção

Dor validada → Seção 3 apresenta a solução

---

### Seção 3: Solução/Abordagem

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #3 - Após validação da dor |
| **Propósito** | Mostrar COMO vocês resolvem diferente |
| **Intensidade Challenger** | 60% |
| **Stakeholder Primário** | Empresário buscando alternativa |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Introduzir o approach único |
| **Exemplo** | "Growth de verdade, sem o teatro corporativo" |
| **Diferenciadores** | 3-4 pilares do approach |
| **Exemplo** | "Foco em receita, não impressões" / "Transparência radical" / "Parceria, não fornecedor" |
| **Visual** | Diagrama simples ou ícones |

#### Contraste Implícito

| Agências Tradicionais | Oitavo Café |
|-----------------------|-------------|
| Contratos longos | Flexibilidade |
| Métricas de vaidade | ROI real |
| Processo opaco | Transparência total |
| Estrutura pesada | Boutique ágil |

#### Stanford Guidelines Atendidos

- #3 Expertise (approach sofisticado)
- #7 Útil (solução clara para o problema)

#### Conexão com Próxima Seção

Approach apresentado → Seção 4 prova que funciona

---

### Seção 4: Prova Social

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #4 - Após solução |
| **Propósito** | PROVAR que o approach funciona |
| **Intensidade Challenger** | 40% |
| **Stakeholder Primário** | Empresário cético querendo evidências |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Número Âncora** | Métrica impressionante |
| **Exemplo** | "R$16M+ em receita gerada para clientes" |
| **Testemunhos** | 2-3 com RESULTADOS ESPECÍFICOS |
| **Formato** | "Nome + Empresa + Resultado numérico + Quote curta" |
| **Exemplo** | "João Silva, CEO TechCo: 'Triplicamos leads qualificados em 4 meses'" |
| **Logos** | Grade de 6-12 logos (se disponíveis) |

#### Regras para Testemunhos

- ✅ Incluir NÚMEROS sempre que possível
- ✅ Resultados de negócio (receita, leads, conversão)
- ✅ Fotos reais dos clientes
- ❌ Elogios genéricos ("ótimo trabalho!")
- ❌ Testemunhos anônimos
- ❌ Vanity metrics (seguidores, likes)

#### Stanford Guidelines Atendidos

- #1 Precisão verificável (números, nomes)
- #4 Pessoas confiáveis (clientes reais)
- #8 Conteúdo atualizado (testemunhos recentes)

#### Conexão com Próxima Seção

Resultados provados → Seção 5 remove última barreira (custo/processo)

---

### Seção 5: Oferta Transparente

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #5 - Após prova social |
| **Propósito** | Remover objeções de custo/processo |
| **Intensidade Challenger** | 40% |
| **Stakeholder Primário** | Empresário pronto para considerar |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Anti-surpresa |
| **Exemplo** | "Sem letras miúdas. Sem surpresas." |
| **Transparência de Preço** | Range ou modelo claro |
| **Exemplo** | "Projetos a partir de R$X" ou "Modelo de precificação transparente" |
| **Processo Visível** | 3-5 etapas simples |
| **Exemplo** | "1. Diagnóstico gratuito → 2. Proposta clara → 3. Execução → 4. Resultados" |
| **Garantia/Compromisso** | Algo que reduz risco |
| **Exemplo** | "Se não entregarmos, você não paga" |

#### Diferenciador Estratégico

Esta seção é onde Oitavo Café **mais se diferencia**:
- 85% das agências NÃO mostram preços
- 40% têm processo opaco
- Transparência = Trust signal poderoso

#### Stanford Guidelines Atendidos

- #2 Organização real (processo real)
- #5 Contato fácil (próximo passo claro)
- #9 Restraint promocional (honestidade sobre custos)

#### Conexão com Próxima Seção

Transparência estabelecida → Seção 6 responde dúvidas restantes

---

### Seção 6: FAQ Estratégico

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #6 - Antes do CTA final |
| **Propósito** | Resolver objeções não-ditas |
| **Intensidade Challenger** | 40% |
| **Stakeholder Primário** | Empresário quase convencido |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Convidativo |
| **Exemplo** | "Perguntas que você provavelmente tem" |
| **Perguntas** | 4-6 questões estratégicas |
| **Tom** | Direto, sem corporativês |

#### Perguntas Sugeridas (Baseadas em Objeções Comuns)

1. **"Por que boutique e não agência grande?"**
   - Resposta: Foco, agilidade, sem overhead

2. **"Quanto tempo leva para ver resultados?"**
   - Resposta: Expectativa realista com números

3. **"E se não funcionar para meu setor?"**
   - Resposta: Diagnóstico gratuito avalia fit

4. **"Vocês fazem X, Y, Z específico?"**
   - Resposta: Escopo claro

5. **"Como funciona o contrato?"**
   - Resposta: Flexibilidade, sem lock-in

#### Stanford Guidelines Atendidos

- #1 Precisão (respostas específicas)
- #7 Útil (resolve dúvidas reais)
- #9 Restraint (honestidade sobre limitações)

#### Conexão com Próxima Seção

Dúvidas resolvidas → Seção 7 converte

---

### Seção 7: CTA Final

| Campo | Conteúdo |
|-------|----------|
| **Posição** | #7 - Final da página |
| **Propósito** | Converter interesse em ação |
| **Intensidade Challenger** | 20% (credibilidade máxima) |
| **Stakeholder Primário** | Empresário pronto para agir |

#### Elementos Obrigatórios

| Elemento | Especificação |
|----------|---------------|
| **Headline** | Chamada para ação clara |
| **Exemplo** | "Pronto para resultados reais?" |
| **CTA Principal** | Mesmo do hero (consistência) |
| **Exemplo** | "Agendar Diagnóstico Gratuito" |
| **Redução de Risco** | Reforço de baixo compromisso |
| **Exemplo** | "30 minutos. Sem compromisso. Insights reais." |
| **Contato Alternativo** | Para quem prefere outro canal |
| **Exemplo** | WhatsApp, email, telefone |

#### Hierarquia de CTA (Recapitulação)

| Nível | Localização | Ação |
|-------|-------------|------|
| Primário | Hero + Seção 7 | "Diagnóstico Gratuito" |
| Secundário | Seção 4-5 | "Ver Cases" ou "Conhecer Processo" |
| Terciário | Footer | Contato geral |

#### Stanford Guidelines Atendidos

- #5 Contato fácil (múltiplas opções)
- #2 Organização real (acessibilidade)

## 10. Guia de Implementação

### Para Designers

| Área | Diretriz |
|------|----------|
| **Hierarquia Visual** | Hero domina, seções progressivamente mais "sérias" |
| **Cores** | Usar paleta Oitavo Café estabelecida |
| **Tipografia** | Headlines bold/provocativos, body profissional |
| **Espaçamento** | Generoso - não sobrecarregar |
| **Mobile-First** | 60%+ do tráfego B2B começa mobile |
| **Above Fold** | Headline + CTA + Trust signals VISÍVEIS sem scroll |
| **Botões** | Min 44x44px para touch |

### Para Redatores

| Área | Diretriz |
|------|----------|
| **Tom Geral** | Irreverente mas confiável |
| **Headlines** | Provocativos, específicos, 6-10 palavras |
| **Body** | Direto, sem jargão, focado em benefícios |
| **CTAs** | Ação-orientados ("Quero X" não "Clique aqui") |
| **Testemunhos** | Números obrigatórios, nomes reais |
| **FAQ** | Respostas curtas, honestas |

### Para Desenvolvedores

| Área | Diretriz |
|------|----------|
| **Performance** | < 3s load time (Core Web Vitals) |
| **LCP** | Hero image otimizada |
| **CLS** | Sem layout shifts |
| **Lazy Loading** | Seções abaixo do fold |
| **Tracking** | Eventos em cada CTA |
| **Schema** | Organization + LocalBusiness |
