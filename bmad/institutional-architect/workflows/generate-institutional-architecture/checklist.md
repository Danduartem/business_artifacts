## Homepage-Specific Validation (if page type is Homepage)

### Above-the-Fold Requirements
- [ ] Hero establishes identity in 5 seconds (who + what + for whom)
- [ ] Headline is SPECIFIC (could NOT apply to competitors)
- [ ] Headline is 6-10 words maximum
- [ ] Value proposition focuses on OUTCOME, not features
- [ ] Single primary CTA in hero (not multiple competing CTAs)
- [ ] Trust signals visible WITHOUT scrolling (logos, numbers, badges)
- [ ] Benefit-focused language ("You get X") not feature-focused ("We do X")

### Anti-Pattern Check
- [ ] NO self-focused messaging ("We are..." instead of "You get...")
- [ ] NO generic headlines that could apply to any company
- [ ] NO multiple competing CTAs in hero section
- [ ] NO content overload / wall of text above fold
- [ ] NO corporate jargon (synergy, leverage, holistic solutions)
- [ ] NO important trust signals buried below fold

### Mobile-First Requirements
- [ ] Designed mobile-first (60%+ of B2B searches start mobile)
- [ ] Touch-friendly buttons specified (min 44x44px)
- [ ] Readable without zooming (16px min font)
- [ ] Fast load time target specified (under 3 seconds)
- [ ] Core Web Vitals considerations noted

### CTA Hierarchy
- [ ] Primary CTA clearly defined (ONE in hero)
- [ ] Secondary CTA defined (for not-ready visitors, after scroll)
- [ ] CTA text is action-oriented ("Get Started", not "Submit")
- [ ] CTA placement and visual treatment specified

### Challenger Brand Check (if applicable)
- [ ] Irreverence targets INDUSTRY, not customer
- [ ] Humor is smart/self-aware, not gimmicky
- [ ] Provocative positioning backed by real proof
- [ ] "Enemy" is status quo, not specific competitors
- [ ] All sections numbered sequentially
- [ ] Each section has complete information (purpose, rationale, trust signals, stakeholder focus)
