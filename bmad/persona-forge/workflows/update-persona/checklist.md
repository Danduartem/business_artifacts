# Update Persona Workflow - Validation Checklist

## Workflow Completion Checklist

### Persona Loading
- [ ] Existing persona document loaded successfully
- [ ] Persona metadata parsed (name, version, dates)
- [ ] Previous version stored for comparison
- [ ] Update type determined (market/qualification/refresh)

### Update Inputs Gathered
- [ ] Update context collected based on type
- [ ] Market research details (if market update)
- [ ] Qualification call results (if qualification update)
- [ ] Refresh context and focus areas (if general refresh)
- [ ] Sufficient information to perform meaningful update

### Update Processing
- [ ] Appropriate agent(s) invoked
- [ ] Market Intelligence Analyst research conducted (if needed)
- [ ] Qualification data analyzed for patterns (if applicable)
- [ ] Updates applied to persona document
- [ ] Core persona identity preserved
- [ ] Revision notes documented
- [ ] Version number incremented appropriately

### Quality Validation
- [ ] Senior Persona Specialist reviewed updates
- [ ] Updates maintain persona coherence
- [ ] Actionability improved (not just changed)
- [ ] Evidence grounding verified
- [ ] Qualification questions updated if needed
- [ ] Validation decision made (approved/refinements)

### Finalization
- [ ] Updated persona document saved with new version
- [ ] Revision history complete
- [ ] Metadata updated (version, date, approval)
- [ ] Update summary created
- [ ] User provided with file location

### Usage Guidance
- [ ] Type-specific guidance provided
- [ ] Team briefing recommendations given
- [ ] Next review date suggested
- [ ] Continuous improvement cycle explained

## Quality Standards Met

### Update Quality
- [ ] Changes grounded in evidence (not speculation)
- [ ] Core persona identity consistent with previous version
- [ ] Updates improve actionability
- [ ] No contradictions introduced
- [ ] Narrative coherence maintained

### Version Tracking
- [ ] Version number follows X.Y format
- [ ] Revision notes explain what, why, and source
- [ ] Confidence levels noted for changes
- [ ] Previous version reference preserved

### Qualification Data Updates (if applicable)
- [ ] Sample size documented
- [ ] Accuracy rate calculated
- [ ] Patterns identified and applied
- [ ] Qualification questions calibrated
- [ ] Confidence levels assigned to changes

### Market Research Updates (if applicable)
- [ ] New sources cited
- [ ] Research findings integrated seamlessly
- [ ] Market position updates reflected
- [ ] Competitive intelligence incorporated
- [ ] Confidence levels noted

## Success Criteria

### Update Effectiveness
- [ ] Persona more accurate after update
- [ ] Actionability increased
- [ ] Evidence supports all changes
- [ ] Team can use updated persona immediately

### Process Efficiency
- [ ] Update completed in reasonable time (30-45 minutes)
- [ ] User felt guided through update process
- [ ] Validation improved quality
- [ ] Version tracking clear and useful
