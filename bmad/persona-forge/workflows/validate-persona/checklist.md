# Validate Persona Workflow - Validation Checklist

## Workflow Completion Checklist

### Persona Loading
- [ ] Persona document loaded successfully
- [ ] Persona format detected (Persona Forge/standard/custom)
- [ ] Metadata parsed (name, version, dates if available)
- [ ] Document structure analyzed
- [ ] Validation type determined (quick/standard/deep)

### Quality Validation (All Types)
- [ ] Senior Persona Specialist validation completed
- [ ] Core structure assessed
- [ ] Completeness checked
- [ ] Issues identified and categorized by severity
- [ ] Recommendations provided

### Standard/Deep Validation Additional Checks
- [ ] Quality score calculated (0-100)
- [ ] Score breakdown by category documented
- [ ] Actionability assessed for go-to-market teams
- [ ] Industry best practices compliance checked
- [ ] Evidence grounding verified
- [ ] Narrative coherence evaluated

### Research Validation (Deep Only)
- [ ] Market Intelligence Analyst research completed
- [ ] Key claims validated against current market data
- [ ] Outdated information identified
- [ ] Unverified assumptions flagged
- [ ] Confidence levels assigned to sections
- [ ] Market context updates provided

### Validation Report
- [ ] Overall assessment provided (Pass/Needs Work/Incomplete)
- [ ] Quality score assigned (if standard/deep)
- [ ] Issues categorized by severity (Critical/Important/Minor)
- [ ] Specific recommendations documented
- [ ] Priority improvements highlighted (top 3-5)
- [ ] Comparison to Persona Forge standards included

### User Review
- [ ] Validation results presented to user
- [ ] User reviewed findings
- [ ] User decision on next steps captured
- [ ] Validation report saved to file

### Next Steps Guidance
- [ ] Guidance appropriate to validation results
- [ ] Critical issues highlighted with context
- [ ] Options for improvement provided
- [ ] Workflow transitions handled (if applicable)

## Quality Standards Met

### Structure and Completeness
- [ ] Core sections identified or flagged as missing
- [ ] Critical information present or gaps documented
- [ ] Document structure evaluated

### Actionability
- [ ] Usability for sales qualification assessed
- [ ] Usability for marketing campaigns assessed
- [ ] Usability for content creation assessed
- [ ] Gaps in actionability identified

### Evidence and Validation
- [ ] Claims evaluated for evidence support
- [ ] Assumptions identified and flagged
- [ ] Research sources noted (if deep validation)
- [ ] Confidence levels assigned (if deep validation)

### Professional Quality
- [ ] Depth and specificity evaluated (not generic)
- [ ] Narrative coherence checked
- [ ] Industry best practices applied
- [ ] Persona Forge standards comparison made

## Validation Type Specific Checks

### Quick Validation
- [ ] Core structure present (demographics, psychographics, pain points, goals)
- [ ] Obvious gaps identified
- [ ] Pass/Fail assessment made
- [ ] Priority issues listed
- [ ] Completed in ~5 minutes

### Standard Validation
- [ ] All Quick Validation checks completed
- [ ] Full quality assessment conducted
- [ ] Quality score calculated (0-100)
- [ ] Detailed recommendations provided
- [ ] Actionability thoroughly assessed
- [ ] Completed in ~10 minutes

### Deep Validation
- [ ] All Standard Validation checks completed
- [ ] Market research validation conducted
- [ ] Current market data referenced
- [ ] Claims validated or flagged
- [ ] Outdated information identified
- [ ] Confidence levels assigned
- [ ] Completed in ~20 minutes

## Issue Severity Validation

### Critical Issues
- [ ] Each critical issue blocks go-to-market effectiveness
- [ ] Each critical issue has clear fix recommendation
- [ ] User warned about impact of critical issues

### Important Issues
- [ ] Each important issue reduces effectiveness (but not blocking)
- [ ] Each important issue has clear improvement path
- [ ] User informed about value of addressing important issues

### Minor Issues
- [ ] Minor issues are truly optional (polish)
- [ ] Minor issues don't block persona usage
- [ ] User aware these are enhancements, not requirements

## Success Criteria

### Validation Effectiveness
- [ ] Validation accurately assessed persona quality
- [ ] Issues identified are legitimate and actionable
- [ ] Recommendations are specific and helpful
- [ ] User has clear understanding of persona quality
- [ ] User knows what to do next

### Process Efficiency
- [ ] Validation completed in expected timeframe
- [ ] User felt guided through validation process
- [ ] Results presented clearly and concisely
- [ ] Next steps obvious and actionable

### Report Quality
- [ ] Validation report is comprehensive
- [ ] Report is well-organized and readable
- [ ] Findings are specific, not generic
- [ ] Recommendations are actionable
- [ ] Report serves as reference for improvements

## Quality Score Validation (Standard/Deep)

### Completeness Score (0-20)
- [ ] All critical sections assessed
- [ ] Scoring reflects actual completeness
- [ ] Missing elements clearly identified

### Depth & Specificity Score (0-20)
- [ ] Generic vs. specific distinction clear
- [ ] Examples evaluated for depth
- [ ] Vague descriptions flagged

### Actionability Score (0-20)
- [ ] Go-to-market usability assessed
- [ ] Qualification readiness evaluated
- [ ] Campaign applicability checked

### Evidence Grounding Score (0-20)
- [ ] Claims vs. assumptions distinguished
- [ ] Research citations evaluated
- [ ] Evidence quality assessed

### Narrative Coherence Score (0-20)
- [ ] Persona unity evaluated
- [ ] Contradictions identified
- [ ] Overall "makes sense" factor assessed

## Research Validation Checks (Deep Only)

### Market Validation
- [ ] Key demographic claims validated
- [ ] Psychographic patterns checked
- [ ] Pain points verified against current market
- [ ] Behavioral patterns confirmed
- [ ] Preference trends validated

### Currency Check
- [ ] Market shifts since creation identified
- [ ] Outdated terminology flagged
- [ ] Changed preferences noted
- [ ] New pain points emerging documented

### Confidence Assessment
- [ ] High confidence claims validated with multiple sources
- [ ] Medium confidence claims have some validation
- [ ] Low confidence claims flagged for user awareness
- [ ] Unverified assumptions clearly marked
